# Автоответчик писем (регистрация и обращение)

## Назначение

- **Письма-обращения** (ящик care): проверка данных в таблице гарантии → ответ клиенту и уведомление админу (данные найдены / не найдены).
- **Письма-регистрации** (ящик warranty): если запись уже есть в таблице регистрации → ответ «спасибо за ещё одну регистрацию», иначе — ответ о приёме первой регистрации.

## Подготовка

1. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. В `.env` указать:
   - `MAIL_USER_CARE`, `MAIL_PASSWORD_CARE` — ящик обращений (куда приходят заявки с формы).
   - `MAIL_USER_WARRANTY`, `MAIL_PASSWORD_WARRANTY` — ящик регистрации гарантии.

   Бот использует **IMAP** (чтение входящих) и **SMTP** (отправка ответов). Для Яндекса обычно нужен **пароль приложения**: Настройки → Безопасность → «Пароли приложений». Включите доступ по IMAP в настройках почтовой программы для обоих ящиков.
   - `TABLE_WARRANTY` — ID таблицы гарантии (из URL: `.../d/ЭТОТ_ID/edit`).
   - `TABLE_REG` — ID таблицы регистрации.
   - `ADMIN_EMAIL` — куда слать уведомления по обращениям.

3. Google Sheets: дать доступ на чтение каждой таблицы сервисному аккаунту (email из `creds.json` → поле `client_email`).

4. Черновики ответов лежат в папке `data/` — их можно править под свой тон.

## Запуск

### Локально (один раз)

```bash
python mail_autoresponder.py
```

### Docker (запуск раз в 2 минуты)

Проверка почты выполняется **каждые 2 минуты**. В корне проекта должны быть `.env` и `creds.json`.

```bash
docker compose up -d --build
```

Остановка: `docker compose down`.

Текущая папка монтируется в контейнер (`.:/app`), поэтому используются ваш `.env`, `creds.json` и папка `data/`; файл `last_check.txt` сохраняется между перезапусками.

Скрипт не использует признак «прочитанности»: обрабатываются только письма, дата которых **новее времени последней проверки**. Время последней проверки хранится в `last_check.txt` с точностью до секунды (UTC) и обновляется после каждого прохода.

## Парсинг писем

Формат писем учтён по примерам из `test_mails/`:

- **Регистрация** (warranty): тема «Новый заказ», в теле — блок «Информация о покупателе» с полями Name, Email, Артикул, Номер_чека, Дата_покупки и т.д.
- **Обращение** (care): тема «Заявка с сайта» / «Request from», в теле — «Содержание заявки» / «Request details» с полями Name, Проблема, Артикул_товара, Номер_чека, дата_с_чека и т.д.

Поиск в таблицах выполняется по артикулу и при возможности по номеру чека (названия столбцов могут быть с пробелами/подчёркиваниями).
